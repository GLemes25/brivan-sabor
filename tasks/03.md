Atue como um Engenheiro de Frontend Sênior e Especialista em UI/UX de Alto Padrão. Tenho um componente Next.js cuja versão mobile está impecável e deve ser preservada. No entanto, a versão Desktop/Tablet precisa ser uma experiência rica, animada e dedicada, não apenas uma versão esticada do mobile.

Sua tarefa é refatorar o código usando Tailwind CSS e a biblioteca `framer-motion`. Você deve manter a estrutura mobile EXATAMENTE igual (preservando as classes base atuais), mas adicionar uma camada de interface premium para o Desktop usando os prefixos `md:`, `lg:` e `xl:`, além de animações de entrada elegantes.

Siga estas diretrizes rigorosamente:

1. Diretiva Client Component (CRÍTICO):
   - Como utilizaremos o `framer-motion`, o arquivo DEVE conter a diretiva `"use client";` na sua exata primeira linha para evitar erros de renderização no servidor do Next.js App Router.

2. Preservação Total do Mobile:
   - Não altere o layout ou comportamento base das classes Tailwind sem prefixo. O mobile continuará com seu design touch-friendly nativo.

3. Layout Estrutural Avançado (Desktop):
   - Envolva as seções em containers para limitar o crescimento infinito (ex: `max-w-7xl mx-auto px-6`).
   - Em listas horizontais (como Categorias ou Destaques), REMOVA o scroll horizontal no desktop. Substitua por um layout em `grid` simétrico ou `flex-wrap` bem espaçado (ex: `md:grid md:grid-cols-4 md:gap-6`). A visualização completa é superior para usuários de mouse.

4. Animações com Framer Motion (Entrada e Scroll):
   - Substitua as tags estáticas principais por componentes animados (ex: `<motion.div>`, `<motion.section>`, `<motion.h2>`).
   - Adicione animações de fade-up sutis e profissionais quando os elementos renderizarem (ex: `initial={{ opacity: 0, y: 20 }} animate={{ opacity: 1, y: 0 }} transition={{ duration: 0.5 }}`).
   - Utilize delays em cascata (`transition={{ delay: 0.1 }}`, `0.2`, etc.) para múltiplos itens (como cards ou botões) aparecerem sequencialmente, criando uma sensação de fluidez e alto padrão.

5. Hover States e Microinterações (O Poder do Desktop):
   - Aproveite a precisão do mouse no CSS. Adicione efeitos de hover elegantes nos elementos interativos (cards, produtos e botões).
   - Use utilitários Tailwind como `hover:-translate-y-1`, `hover:scale-105`, `transition-all duration-300 ease-out`.
   - Adicione sombras e glows sutis em dourado no hover (ex: `hover:shadow-lg hover:shadow-brand-gold/20`).
   - Utilize a classe `group` no container pai e `group-hover:` nos elementos filhos para criar interações ricas (ex: passar o mouse no card e o ícone interno mudar de cor).

6. Tipografia e Proporção Fluida:
   - Escale a tipografia muito pequena para uma leitura confortável no desktop (ex: de `text-[10px]` para `md:text-sm lg:text-base`).
   - Escale o tamanho dos ícones do `lucide-react` proporcionalmente nos breakpoints maiores.

7. Arquitetura e Regras de Código:
   - Código, variáveis, funções e imports ESTRITAMENTE em INGLÊS.
   - Textos de interface ESTRITAMENTE em Português do Brasil (pt-BR).
   - Retorne o código limpo, elegante e SEM COMENTÁRIOS dentro do arquivo.
